# GAMES MASTER — Texture Style Guide

## CRITICAL OUTPUT REQUIREMENTS

Every texture in this project must satisfy all of the following. Violating any one of these
makes the output unusable:

1. **64 × 64 pixels exactly. Square. 1:1 aspect ratio.** Width = 64. Height = 64. Not 128.
   Not 256. Not rectangular. The raycaster hard-codes 64-pixel UV coordinates.
2. **Pixel art.** Not painted. Not photorealistic. Not watercolour. Not impressionist.
   Every pixel is a flat, aliased, single-colour square. No anti-aliasing. No smooth
   gradients. No brush strokes. No blur. Hard edges everywhere.
3. **PNG, 8-bit RGB, no alpha channel.**
4. **No post-processing filters.** No vignette, no bloom, no lens flare, no noise filter.

**Style reference:** Early 1990s first-person shooter game wall textures.
Hand-pixeled retro game tile assets. Each pixel individually placed, not generated by a
painting algorithm. Looks like a texture ripped from a 1993 shareware FPS, not a painting.

---

## What these textures are for

A first-person raycasting horror game (Wolfenstein / DOOM renderer style).
The engine casts one ray per screen column. When a ray hits a wall it reads a single
vertical column of pixels from the wall texture. Floor and ceiling pixels are computed
per screen pixel using planar projection. All textures must be pixel art to match the
renderer's visual style and to sample correctly at integer UV coordinates.

---

## Technical specification

### Wall textures

- **Size: 64 × 64 pixels. Square.**
- Sampled as a single vertical column (texX 0–63). The column is stretched vertically to
  fill the on-screen wall strip height.
- **Horizontal tiling must be seamless** — left and right edges must match exactly.
- Vertical seamlessness optional.
- Format: PNG, 8-bit RGB, no alpha.

### Floor and ceiling textures

- **Size: 64 × 64 pixels. Square.**
- Tiled across the floor/ceiling plane using planar perspective mapping.
- **Both axes must tile seamlessly.**
- Format: PNG, 8-bit RGB, no alpha.

### Pixel art specification

- **Every pixel is a deliberate, aliased, flat-colour square.** No sub-pixel rendering.
- Pixel clusters of 2–4 px wide represent one surface feature (a mortar joint, a grain
  line, a velvet fold highlight).
- Shading is **banded** — discrete value steps with hard edges between them. No gradients.
- Colour palette per texture: 12–20 distinct colours maximum.
- No anti-aliasing on any edge. No smoothing of any kind.
- The visual result must look like a hand-pixeled retro game tile, not a painting.

## Setting and aesthetic

Abandoned Victorian Opera House — built 1870s, sealed 1940s, untouched since.
Grand in its bones, hostile now: damp, dark, rotting.

**Materials:**

- **Granite** — structural walls. Dark, coarse-grained, crystalline igneous rock.
  Heavy, permanent, cold. Salt-and-pepper speckle visible even at dark values.
- **Wood flooring** — wide-plank oak or pine. Grained, worn, stained. Not parquet.
- **Velvet** — heavy theatrical velvet, stage curtain walls, deep crimson.
  Directional pile: fold ridges lighter, fold creases near-black.
- **Plasterwork** — auditorium walls, lime plaster, once ornate, now crumbling.

**Atmosphere:** dank, cold, musty, mineral. Mould, salt, rot, oxidised metal.

**Colour temperature:**

- Floors 0–1 (Foyer, Auditorium): very dark amber-grey. Warmer shadows.
- Floors 2–3 (Gallery, Flies): very dark blue-grey. Colder, more industrial.
- No surface bright. Maximum value 55–60% across all tiles.
